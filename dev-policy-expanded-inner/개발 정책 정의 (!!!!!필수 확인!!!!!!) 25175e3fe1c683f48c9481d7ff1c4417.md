## 4. 예외처리

[detail](https://www.notion.so/detail-bdd75e3fe1c68286b4b281b595ef408c?pvs=21)

예외 발생 → enum 선택 → errorResponse 클래스 구조 확인(메서드 체이닝 방식으로 지정했습니다.) → 각 변수에 넘길 값 지정(4개입니다 확인 잘 해주세요.)

- 컨트롤러나 서비스에서 개별적으로 `try-catch`로 응답을 말아주는 것 자제 부탁드립니다. → globalExceptionHandler 활용 부탁 대표적인 에러 enum설정 해놨고 필요하면 개별 추가 가능합니다.
- 필요시 RuntimeException 상속 Exception 클래스 생성하시면 됩니다. 일단 초기 환경이라 그냥 BusinessException 클래스 만들어 놨습니다. → 방법은 기존 예외처리 클래스 활용하거나 새로운 예외처리 클래스 만들어 거기로 throw 하는 식 
ex ) (현재 초기 파일) 비즈니스 로직 예외는 BusinessException 클래스 형태로 throw 하면(globalExceptionHandler가 받음) 됩니다.
- enum에 공통 예외와 도메인별 예외 상황 코드로 분리하여 추가해야 합니다.
- 방어벽 구분
    - **1차 방어:** `@Valid` 등을 이용한 입력값 검증.
    - **2차 방어:** 서비스 로직 내 `if`문을 통한 비즈니스 조건 검증.
    - **비즈니스 예외:** 검증 실패 시 `BusinessException`을 던져 즉시 로직 중단.

## 5. 테스트 코드 작성 (예외 위주 테스트)

- Repository 테스트 → JPA 쿼리와 DB 연동만 빠르게 테스트
- Service 테스트 → 의존하는 Repository 등은 반드시 `@Mock`으로 가짜 객체를 만들어 주입하고, 순수 비즈니스 로직만 검증
- Controller 테스트 → HTTP 요청/응답 변환, Validation 검증, 상태 코드 반환 등을 MockMvc를 통해 검증

## 6. Security 설정

- 현재 원활한 API 테스트를 위해 CSRF와 엔드포인트 권한 제한을 모두 풀어두었습니다. Postman으로 자유롭게 테스트하세요. (추후 JWT 도입 시 잠글 예정입니다.)
